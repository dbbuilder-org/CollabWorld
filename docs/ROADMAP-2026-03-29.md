## Key Notes

> **Prisma migrations are missing** — The schema was always `db push`'d, never `migrate dev`'d. The `prisma/migrations/` directory is empty. This is fine for active development but before 1.0 production launch you should generate migration files so `migrate deploy` can run deterministically in CI.

> **R2 credentials** — `R2_ACCESS_KEY_ID` and `R2_SECRET_ACCESS_KEY` are in `.env.local` but not in `packages/db/.env` or Render. Avatar upload will fall back to local disk on Render until these are added.

> **`packages/db/.env`** — Created to let Prisma CLI find `DATABASE_URL` locally. Must be kept in sync with `.env.local`. Not committed (gitignored). If you clone fresh, recreate it.
